# Operator Journal — Session Log

Date: 2026-02-01  
Focus Area: Microsoft Entra Conditional Access  

---


## 1️⃣ Objective (What was I trying to control or prove?)
Establish a baseline Conditional Access policy that enforces MFA for all users when signing in from outside a trusted geographic location (NZ-Trusted), while preserving administrative break-glass access and validating behavior in Report-only mode before enforcement.

---

## 2️⃣ Change Made (What did I actually touch?)
- System: Microsoft Entra ID
- Control Type: Conditional Access Policy
- Policy Name: MFA-Outside-NZ-Trusted
- Scope:
  - Users: All users included
  - Exclusions: PVD-GlobalAdmin (break-glass safety account)
  - Resources: All cloud apps
  - Conditions: Location-based (Any location, excluding NZ-Trusted)
  - Grant Control: Require Multi-Factor Authentication
  - State: Report-only

---

## 3️⃣ Test Performed (How did I validate it safely?)
- Used Conditional Access "What If" tool to simulate:
  - User: PVD-GlobalAdmin
  - IP Address: 8.8.8.8 (Non-NZ location)
  - Device Platform: Windows
  - Client App: Browser
- Verified that the policy appears as "Would apply" in simulation without enforcing access restrictions.
- Reviewed Sign-in Logs to confirm Conditional Access evaluation field shows policy as Not Applied (Report-only mode).

---

## 4️⃣ Recovery Path (If this broke, how would I get back in?)
- Break-glass account excluded from policy.
- Global Admin access retained via trusted admin identity.
- Policy can be disabled or deleted via Entra Admin Center → Conditional Access → Policies.
- Fallback: Use Microsoft emergency access account if tenant lockout occurs.
